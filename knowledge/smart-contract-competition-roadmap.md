# 智能合约竞赛考点学习路径

## 适用目标

本模块用于训练区块链应用类竞赛中的智能合约开发能力。内容来自对训练材料中通用考点的抽象整理，只讲知识点、代码模式和答题方法。示例仅使用通用业务模型，不包含非公开训练材料细节。

智能合约部分的核心不是“会写几个函数”，而是能把业务状态、权限控制、链上存储、事件记录、安全检查组织成可验证的合约逻辑。

```text
业务实体 -> 存储结构 -> 权限规则 -> 状态流转 -> 事件记录 -> 测试验证
```

## 能力分层

| 层级 | 必学知识点 | 典型考法 |
| --- | --- | --- |
| 基础语法 | 类型、函数、可见性、返回值 | 补变量声明、函数签名、返回字段 |
| 数据建模 | `struct`、数组、`mapping` | 设计记录、按编号查询、判断是否存在 |
| 权限控制 | `owner`、角色、`modifier`、`require` | 限制指定账户调用函数 |
| 状态流转 | 状态码、枚举、布尔标记 | 防重复、审核、转移、完成状态 |
| 事件记录 | `event`、`emit` | 操作后写链上日志，供后端解析 |
| 字符串处理 | `abi.encodePacked`、字段拼接 | 拼接多字段记录或调用参数 |
| 合约间调用 | 接口、地址、`call`、返回值检查 | 控制合约调用存储合约 |
| 存储分离 | Storage / Controller 分层 | 业务逻辑和数据存储拆开 |
| 安全修复 | 重入、权限缺失、整数边界 | 识别漏洞并补防护 |
| 测试验证 | Truffle/Hardhat 测试 | 部署、调用、断言状态 |

## 从易到难的学习顺序

### 1. Solidity 基础语法

先掌握变量、函数、返回值、可见性。

必会内容：

- `uint256`、`string`、`address`、`bool`
- `public`、`private`、`internal`、`external`
- `view`、`pure`
- `memory`、`storage`
- `returns`

训练目标：

> 能补全一个查询函数，返回指定记录中的多个字段。

### 2. 业务数据建模

合约题经常先给业务字段，再要求补 `struct`、`mapping` 和数组。

必会内容：

- `struct Record`
- `mapping(uint256 => Record)`
- `mapping(address => bool)`
- `uint256[] ids`
- 存在性判断

训练目标：

> 能用 `mapping` 保存一条记录，并通过编号查询。

### 3. 权限控制

权限是智能合约最常见失分点。只要函数涉及新增、修改、审核、授权，就应该先判断调用者身份。

必会内容：

- `owner`
- `msg.sender`
- `require`
- `modifier onlyOwner`
- 角色映射

训练目标：

> 能补全一个只有管理员可调用的新增角色函数。

### 4. 状态流转

业务上链后通常存在“待处理、已完成、已拒绝、已失效”等状态。

必会内容：

- 状态码常量
- `enum`
- 防重复提交
- 状态前置条件
- 状态更新

训练目标：

> 能补全一个审核函数：先判断记录存在，再判断当前状态，最后修改状态。

### 5. 事件记录

事件是后端监听链上结果的重要入口。合约状态改变后应 `emit` 事件。

必会内容：

- 定义 `event`
- `indexed`
- `emit`
- 事件参数设计

训练目标：

> 能在新增、修改、审核后触发事件，事件中包含操作者、记录编号、状态和时间。

### 6. 存储与控制分离

较难材料中常见 Storage 合约保存数据，Controller 合约负责校验和调用。

必会内容：

- 存储合约地址
- 控制合约持有 Storage 实例
- 合约间函数调用
- 统一返回码
- `require(success, "...")`

训练目标：

> 能补全一个 Controller 函数：拼接字段，调用 Storage，检查返回结果，触发事件。

### 7. 字段数组与字符串拼接

部分链上存储会把多字段业务数据转成字符串或字段数组。

必会内容：

- `string[] memory`
- `abi.encodePacked`
- 字段顺序
- 字符串比较
- 类型转换

训练目标：

> 能把多个字段按指定顺序拼成链上存储字符串，并在查询时按字段位置还原。

### 8. 合约安全

安全题不是只背漏洞名，而是要能定位危险代码并补防护。

必会内容：

- 权限缺失
- 重入风险
- 外部调用返回值检查
- 状态先改后调用
- 输入边界检查
- 防重复记录

训练目标：

> 能指出一个函数为什么可被越权调用，并用 `modifier` 或 `require` 修复。

### 9. 合约测试

测试题常考部署合约、调用函数、断言返回值或状态变化。

必会内容：

- 获取合约实例
- 调用写函数
- 调用读函数
- 断言状态
- 捕获 revert

训练目标：

> 能编写一个测试：新增记录成功、重复新增失败、非授权账户调用失败。

## 学习方法

每次练合约题，按这个顺序拆：

1. 字段是什么。
2. 谁能调用。
3. 调用前要检查什么。
4. 状态如何变化。
5. 是否要触发事件。
6. 后端或测试如何验证。

不要直接写函数体。先把存储结构和状态规则定清楚，代码才不会散。
