# 前端竞赛考点学习路径

## 定位

本知识库以前端训练为主，面向 Vue3 项目级挖空补全。内容由学长结合比赛经验、训练复盘和备赛建议整理，帮助学员掌握项目阅读方法、补全套路和限时答题节奏。

这类前端题不是单纯考 Vue 语法，而是考选手能否在已有工程中快速补齐页面链路：

```text
识别页面空缺 -> 读变量和组件 -> 查 API 映射 -> 调统一请求 -> 更新页面状态 -> 展示结果和证据
```

## 前端优先级

| 优先级 | 能力 | 为什么重要 |
| --- | --- | --- |
| P0 | Vue3 项目阅读 | 先知道空缺在哪、已有变量在哪、接口怎么调 |
| P0 | 大块函数补全 | 登录、列表、分页、下载、详情常常空一整段 |
| P0 | 模板区域补全 | 卡片头、卡片体、卡片尾、详情区域可能整块缺失 |
| P1 | 统一请求封装 | 比赛项目通常不是裸 axios，而是 API 编号或统一 Fetch |
| P1 | 状态反馈 | loading、empty、message、异常分支会直接影响评分观感 |
| P1 | 路由和 token | 登录成功、刷新保持、接口鉴权是前端闭环起点 |
| P2 | 后端/链端理解 | 只需要理解接口字段、链上凭证、部署结果，不作为前端主线 |
| P3 | Vue2 补充 | 主要作为省赛或旧项目补充学习，不放在本赛主线 |

## 项目阅读顺序

拿到 Vue3 挖空工程后，不要马上填代码。先按这个顺序读：

1. `package.json`：确认 Vue3、Vite、UI 库、请求库、状态库。
2. `src/main.ts`：确认全局插件、样式、路由、状态管理。
3. `src/router`：确认页面入口、登录页、业务页、权限字段。
4. `src/fetch` 或 `src/api`：确认统一请求、API 编号、响应结构。
5. `src/store`：确认用户信息、token、角色、全局 loading。
6. `src/views`：定位 `选手填写部分`，判断是模板空还是函数空。
7. `src/components`：复用已有弹窗、表单、卡片、空状态，不要重复造组件。

## 挖空类型

| 类型 | 常见空法 | 解题动作 |
| --- | --- | --- |
| 模板属性空 | `:key=""`、`:class="选手填写部分"` | 从循环变量或状态变量中取表达式 |
| 模板整块空 | `<!-- 选手填写部分 -->` | 复用已有卡片/按钮/字段样式补一段区域 |
| 表单字段空 | `v-model:value="选手填写部分"` | 找 `reactive` 表单对象和字段名 |
| 事件空 | `选手填写部分` 放在按钮上 | 补完整 `@click="方法名"` |
| 登录逻辑空 | 函数体整段为空 | 校验、请求、存 token、拉用户、跳转 |
| 列表逻辑空 | 获取列表函数为空 | loading、组参数、调用 API、赋 records/total、catch |
| 下载逻辑空 | 下载函数为空 | 生成 Blob、文件名兜底、释放 URL、message |
| 联调空 | API 编号、参数、响应字段缺失 | 查 API 映射和接口文档，保持前后端字段一致 |

## 从易到难

### 1. 模板小空

训练目标：

- 会补 `:key`、`{{ }}`、`:class`、`v-if`。
- 不改变已有结构，只填空位。
- 不把循环变量写丢。

### 2. 组件区域补全

训练目标：

- 在已有卡片、表格、弹窗里补一块内容。
- 使用已有 class 和组件风格。
- 能根据字段展示状态、按钮和操作入口。

### 3. 表单与弹窗

训练目标：

- 会声明 `reactive` 表单。
- 会绑定 `v-model:value`。
- 会做非空校验和错误提示。
- 提交成功后关闭弹窗并刷新列表。

### 4. 登录与鉴权

训练目标：

- 账号密码非空校验。
- 调统一请求方法。
- 成功后保存 token。
- 获取当前用户信息。
- 跳转首页或业务页。

### 5. 列表、分页、搜索

训练目标：

- 查询前设置 loading。
- 传当前页、每页数量、搜索条件。
- 响应后赋值列表和总数。
- 失败时提示并保持状态清晰。

### 6. 文件上传和下载

训练目标：

- 上传前校验文件类型和大小。
- 使用 `FormData`，不要把大文件长期转 Base64。
- 下载使用 Blob，并及时 `URL.revokeObjectURL()`。
- 文件名要有兜底。

### 7. 链上结果展示

训练目标：

- 把交易哈希、区块高度、合约地址、状态等作为页面证据展示。
- 列表、详情、结果弹窗都要能看见关键字段。
- 不只在控制台打印。

### 8. 交付与自检

训练目标：

- 页面可运行。
- 登录、查询、提交、详情、下载至少跑通核心闭环。
- 截图能证明功能完成。

## 比赛答题策略

### 先做能闭环的空

优先顺序：

1. 登录函数。
2. 列表查询函数。
3. 模板区域显示。
4. 搜索和分页。
5. 下载、详情、状态展示。
6. 样式微调。

### 大块函数统一模板

```ts
async function runAction() {
  if (前置条件不满足) {
    message.error('请先补全必要信息')
    return
  }

  loading.value = true
  try {
    const res = await Fetch({
      uriCode: 'GENERIC_API_CODE',
      ...请求参数
    })
    页面状态.value = res.data
    message.success('操作成功')
  } catch (error) {
    message.error('操作失败')
  } finally {
    loading.value = false
  }
}
```

### 模板整块补全顺序

```text
先补标题/状态 -> 再补字段展示 -> 再补按钮事件 -> 最后补空状态或兜底
```

### 不要犯的错误

- 不要改动题目已经给出的循环结构。
- 不要凭感觉新建一套请求方法。
- 不要写死接口地址，先看工程是否已有 API 映射。
- 不要只 `console.log`，页面必须有反馈。
- 不要把大文件转成多个 Base64 副本。
- 不要把 Vue2 写法混到 Vue3 `<script setup>` 里。

## Vue2 的位置

Vue2、Element UI、Vue CLI 更适合作为省赛或旧项目补充学习。本知识库保留这部分内容，但不把它作为金砖前端主线。主线训练统一围绕 Vue3、Vite、TypeScript、组合式 API 和项目级挖空补全。
