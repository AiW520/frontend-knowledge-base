# 前端竞赛考点学习路径

## 适用目标

本模块用于训练区块链应用类竞赛中的前端开发能力。内容只抽象讲解通用知识点与答题方法。示例仅使用通用业务模型，不包含非公开训练材料细节。

前端部分通常不是考“页面好不好看”，而是考选手能否在限定时间内完成一条稳定的数据链路：

```text
读懂业务字段 -> 编写页面交互 -> 调用后端接口 -> 展示业务结果 -> 展示链上凭证 -> 截图交付
```

## 能力分层

| 层级 | 必学知识点 | 典型考法 |
| --- | --- | --- |
| 基础语法 | Vue 模板、`v-model`、`v-for`、`v-if`、事件绑定 | 在指定位置补全页面绑定与按钮事件 |
| 表单交互 | 表单模型、校验规则、提交状态、错误提示 | 登录、注册、新增、审核、查询条件 |
| 接口调用 | axios、baseURL、请求参数、响应处理 | 根据接口完成查询、提交、更新 |
| 鉴权状态 | token、localStorage、cookie、请求拦截器 | 登录后携带身份访问业务接口 |
| 路由组织 | Vue Router、角色入口、页面跳转、守卫 | 登录成功后进入对应功能页 |
| 数据展示 | 表格、分页、详情、状态标签、空状态 | 展示业务列表、操作记录、查询结果 |
| 链上凭证 | 交易哈希、区块高度、合约地址、上链状态 | 把接口返回的链上信息展示为可评分证据 |
| 文件能力 | 上传、下载、预览、Blob、FileReader | 处理材料、证明、附件、签名文件 |
| 可视化 | ECharts、统计卡片、趋势图 | 展示区块、交易、业务统计 |
| 工程化 | API 封装、环境变量、代理、错误边界 | 提升答题速度和代码稳定性 |

## 从易到难的训练顺序

### 1. 页面绑定与事件

先掌握最小闭环：输入框能绑定数据，按钮能触发方法，页面能显示结果。

必会内容：

- `v-model` 绑定输入框。
- `@click` 绑定按钮。
- `v-if` / `v-show` 展示状态。
- `v-for` 渲染数组。

训练目标：

> 给定一个表单页面，补全输入绑定、提交事件和成功提示。

### 2. 表单校验与提交

表单题是最常见的前端题型。不要只把请求发出去，还要保证数据有效、状态清楚。

必会内容：

- 必填校验。
- 长度和格式校验。
- 提交中 loading。
- 成功/失败 message。
- 提交后重置或刷新。

训练目标：

> 完成一个新增表单：校验通过后调用接口，成功后关闭弹窗并刷新列表。

### 3. axios 接口调用

接口调用是前端得分核心。选手要能看懂接口参数、响应结构，并把响应数据映射到页面。

必会内容：

- `axios.get` 查询。
- `axios.post` 提交。
- 请求参数位置：query、body、path。
- 响应结构判断：`code`、`data`、`message`。
- 异常捕获。

训练目标：

> 根据接口返回的数组数据，渲染到表格，并处理空数据和加载失败。

### 4. 登录状态与请求拦截器

登录不是结束，而是后续业务接口的起点。

必会内容：

- 登录成功保存 token。
- 请求时自动携带 token。
- token 失效后跳转登录页。
- 用户信息持久化。

推荐写法：

```ts
request.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = token
  }
  return config
})
```

训练目标：

> 登录后进入业务页面，刷新页面仍能携带 token 查询用户信息。

### 5. 路由与角色页面

竞赛项目经常存在多个角色。前端必须能根据身份进入不同功能页。

必会内容：

- 路由表配置。
- `router.push()` 编程式跳转。
- 路由参数。
- 路由守卫。
- 角色菜单过滤。

训练目标：

> 登录成功后，根据用户角色跳转到对应首页，并控制菜单显示。

### 6. 表格、分页与详情

业务数据通常通过表格展示。表格不是把数组丢进去就结束，还要让状态清晰。

必会内容：

- `el-table` / 普通表格渲染。
- 状态标签。
- 操作按钮。
- 分页查询。
- 详情弹窗。
- 删除或更新后的列表刷新。

训练目标：

> 完成一个业务列表页：支持查询、分页、查看详情、状态展示。

### 7. 链上凭证展示

区块链应用前端必须展示链上证据。接口返回的交易哈希、区块高度、合约地址、上链状态，不应藏在控制台里。

必会内容：

- 交易哈希展示。
- 区块高度展示。
- 合约地址展示。
- 上链状态标签。
- 溯源/流转记录时间线。

推荐结构：

```ts
interface ChainProof {
  txHash: string
  blockNumber?: number
  contractAddress?: string
  status: 'pending' | 'success' | 'failed'
}
```

训练目标：

> 在业务提交成功后展示链上凭证区域，裁判只看页面也能判断操作已经上链。

### 8. 文件上传、预览与下载

文件题要注意性能和内存。除非题目明确要求，不要把大文件完整转成 Base64 长字符串长期保存在内存中。

必会内容：

- 文件选择。
- 文件大小和类型校验。
- `FormData` 上传。
- Blob 下载。
- 图片预览。
- 释放对象 URL。

推荐写法：

```ts
const previewUrl = URL.createObjectURL(file)

onBeforeUnmount(() => {
  URL.revokeObjectURL(previewUrl)
})
```

训练目标：

> 完成一个附件上传和下载功能，上传前校验文件类型，下载失败时给出提示。

### 9. 数据可视化

可视化不是装饰，必须服务业务判断。

必会内容：

- 统计卡片。
- 折线图、柱状图。
- ECharts 初始化和销毁。
- 数据变化后更新图表。
- 空数据展示。

训练目标：

> 根据接口返回的统计数据，绘制业务数量趋势或链上交易趋势。

### 10. 交付与截图

前端最终要服务评分。页面功能做完后，要能形成可核验的交付证据。

建议截图覆盖：

- 登录成功状态。
- 表单填写状态。
- 提交成功提示。
- 列表刷新后的结果。
- 详情页数据。
- 链上凭证信息。
- 异常处理或空状态。

## 学习方法

每个知识点都按同一套步骤训练：

1. 先看页面需要什么数据。
2. 再看接口需要什么参数。
3. 然后写响应式变量。
4. 再写请求方法。
5. 最后把状态展示到页面。

不要只背代码。真正需要形成的是“读题到代码”的转换能力。
